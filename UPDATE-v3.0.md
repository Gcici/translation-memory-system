## 🔧 配置选项

### 优先级设置

**普通优先级:**

**紧急优先级:**

**付费版($19/月):**
- 每月50次
- 普通优先级

**专业版($49/月):**
- 无限次数

## 📊 数据结构

### translation_requests 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 请求ID |
| user_id | UUID | 用户ID |
| japanese_text | TEXT | 日文原文 |
| context | TEXT | 上下文说明 |
| priority | VARCHAR | 优先级 |
| status | VARCHAR | 状态 |
| human_translation | TEXT | 翻译结果 |
| translator_id | UUID | 客服ID |
| rating | INTEGER | 用户评分 |
| feedback | TEXT | 用户反馈 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| completed_at | TIMESTAMP | 完成时间 |

### 状态值

- `pending` - 待处理
- `processing` - 处理中
- `completed` - 已完成
- `cancelled` - 已取消

## 🔒 安全特性

### 数据隔离

✅ **行级安全(RLS)**
- 用户只能看自己的请求
- 管理员可以看所有请求
- PostgreSQL底层保护

✅ **权限控制**
- 创建请求需要登录
- 处理请求需要管理员权限
- 评分只能评自己的请求

### 数据保护

- 自动加密存储
- HTTPS传输
- 定期自动备份
- 审计日志记录

## 🆘 常见问题

### Q1: 需要额外的服务器吗?

**A:** 不需要!
- 使用现有Supabase数据库
- 无需额外配置
- 完全免费(在免费额度内)

### Q2: 需要招聘多少客服?

**A:** 取决于请求量:
- 每位客服处理: 20-30个/天
- 估算公式: 客服数 = 日请求量 / 25
- 建议从2-3人开始

### Q3: 如何收费?

**A:** 灵活定价:
- 按次收费: $5-10/次
- 订阅制: $19-49/月
- 企业定制: 按需报价

### Q4: 可以外包客服吗?

**A:** 可以!
- 使用翻译外包平台
- 自由职业译者
- 翻译公司合作
- 注意质量控制

### Q5: 如何防止刷请求?

**A:** 多重保护:
- 设置每日请求上限
- 需要登录才能请求
- 监控异常行为
- 封禁滥用账号
